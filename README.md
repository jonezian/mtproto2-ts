# mtproto2-ts

Complete TypeScript implementation of Telegram's MTProto 2.0 protocol.

![Build Status](https://img.shields.io/badge/build-passing-brightgreen)
![License](https://img.shields.io/badge/license-MIT-blue)
![Node](https://img.shields.io/badge/node-%3E%3D22-green)

## Features

- **Type-safe** -- strict TypeScript throughout, no `any` types
- **Modular monorepo** -- 7 independent packages under the `@mtproto2/` scope
- **Zero runtime dependencies** -- only Node.js built-in modules (`node:crypto`, `node:net`, `node:events`)
- **Node.js 22+** -- built for modern runtimes, ESM-only
- **974+ tests** -- comprehensive test coverage across 59 test files
- **Auto schema updates** -- pipeline to fetch, diff, and regenerate TL types from upstream

## Quick Start

```bash
npm install @mtproto2/client @mtproto2/crypto
```

```ts
import { TelegramClient, MemorySession } from '@mtproto2/client';

const client = new TelegramClient({
  apiId: 12345,
  apiHash: 'your_api_hash',
  session: new MemorySession(),
});

await client.connect();
const me = await client.getMe();
await client.disconnect();
```

| [`@mtproto2/crypto`](packages/crypto/) | Cryptographic primitives (AES-IGE, RSA-PAD, DH, PQ) | Stable |
| [`@mtproto2/tl-schema`](packages/tl-schema/) | TL schema parser and TypeScript code generator | Stable |
| [`@mtproto2/tl-types`](packages/tl-types/) | Auto-generated TypeScript types for Telegram API (layer 216) | Generated |

## Architecture

```
@mtproto2/client
    |
    v
@mtproto2/mtproto
    |
    +-----> @mtproto2/transport --> @mtproto2/crypto
    |                          --> @mtproto2/binary
    +-----> @mtproto2/crypto
    +-----> @mtproto2/binary
    +-----> @mtproto2/tl-types

@mtproto2/tl-types  <---(generated by)---  @mtproto2/tl-schema
```

- **binary** and **crypto** are leaf packages with zero internal dependencies.
- **tl-schema** is a build-time tool that generates **tl-types**.
- **transport** depends on **binary** and **crypto** for framing and obfuscation.
- **mtproto** is the core engine, combining transport, crypto, and types.
- **client** provides a high-level API on top of the mtproto engine.

## Security

All cryptographic operations use Node.js `crypto` built-in module:

- Random generation exclusively via `crypto.randomBytes()` -- `Math.random()` is never used
- Timing-safe comparisons via `crypto.timingSafeEqual()` for msg_key and nonce verification
- Full DH parameter validation (2048-bit safe prime check, subgroup validation, g_a range check)
- Padding validation on decryption (12--1024 bytes enforced)
- Buffer size limits to prevent memory exhaustion attacks

## Schema Updates

The TL schema can be updated from upstream Telegram sources:

```bash
npm run fetch-schema     # Download latest api.tl and mtproto.tl
npm run diff-schema      # Show changes between current and new schema
npm run generate         # Regenerate TypeScript types
npx vitest run           # Verify nothing is broken
```

This pipeline ensures the type definitions stay current with Telegram API updates.
